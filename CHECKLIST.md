Repo polish checklist

- Description: one-line value prop and tech stack badges.
- README: problem, solution, architecture diagram, setup/run, screenshots, roadmap.
- Topics: add searchable tags (e.g., computer-vision, yolo, mern, express, mongodb).
- License: choose a license (MIT included in this repo as a default template).
- CI: enable basic tests/lint on push and PR (see templates under `TEMPLATES/.github/workflows/`).
- Issues: use `bug_report` and `feature_request` templates (provided under `TEMPLATES`).
- PRs: include checklist and testing steps (see `TEMPLATES/.github/pull_request_template.md`).
- Code style: Prettier/ESLint for JS/TS; Black/Ruff for Python. Pin versions.
- Security: `SECURITY.md` for vulnerability reporting; review Dependabot alerts.
- Visibility: pin 6 best repos on your profile; add demo GIFs/screenshots.
